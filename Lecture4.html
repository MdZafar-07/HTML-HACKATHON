üìÅ File Path System

The Fundamental Truth

A website is not one single file. It's a collection of different files (HTML, CSS, JavaScript, images, videos, fonts) that all need to work together.

Relative File Paths

Relative file paths are used to link files within the same project or directory structure. They are based on the current location of the file that contains the link.Relative paths give directions to a file starting from the location of the file you are currently in.

In index.html, to link to styles.css:
<link rel="stylesheet" href="css/styles.css">		
In about.html, to link to logo.png:
<img src="images/logo.png" alt="Logo">
In contact.html, to link to script.js:
<script src="js/script.js"></script>
Symbol Meaning Example
. Current Directory ./css/styles.css
.. Parent Directory (one level up) ../images/logo.png
/ Root Directory (top-level directory) /css/styles.css

Key Point:

Use ../ to go up one level in the folder hierarchy. Each ../ moves up one folder.

Absolute File Paths

Absolute paths give the full URL to a resource on the web. Use these for external resources

Example:

	
< a href="https://www.google.com">Search on Google</a>

Warning:

Never use absolute paths for your own files (like C:/Users/YourName/website/image.png). These will break when you upload your site to a server!


üìí HTML Boilerplate

What is Boilerplate

Boilerplate code is a standard template that provides the basic structure and essential elements needed to start a web page. It saves time and ensures consistency across different web pages.

Why We Need Boilerplate Code?

Browsers need specific instructions to properly render your webpage. The boilerplate provides this essential "setup information"..

Basic HTML5 Boilerplate:

<!DOCTYPE html>
<html lang="en">	
<head>
	<meta charset="UTF-8">	
	<meta name="viewport" content="width=device-width, initial-scale=1.0">
	<title>Document Title</title>	
</head>
<body>	
</body>
</html>

Explanation of Each Part:

Element Description
<!DOCTYPE html> Declares the document type and version of HTML (HTML5 in this case).
<html lang="en"> Root element of the HTML document. The lang attribute specifies the language of the document.
<head> Contains meta-information about the document, such as character encoding, viewport settings, and the title.
<meta charset="UTF-8"> Sets the character encoding for the document to UTF-8, which supports most characters from all languages.
<meta name="viewport" content="width=device-width, initial-scale=1.0"> Ensures the webpage is responsive by setting the viewport to match the device's width and initial zoom level.
<title>Document Title</title> Sets the title of the webpage, which appears in the browser tab and is important for SEO.
<body> Contains the content of the webpage that will be displayed to users, such as text, images, links, etc.

Using this boilerplate code as a starting point will help you create well-structured and standards-compliant web pages.

Pro Tip:

Always include the viewport meta tag to ensure your website looks good on mobile devices


üåê Page Structure with Semantic HTML

What is Semantic HTML?

Semantic HTML uses HTML5 elements that clearly describe their meaning in a human- and machine-readable way. This improves accessibility, SEO, and code maintainability.

Why Semantic HTML Matters

Semantic HTML helps browsers, search engines, and assistive technologies understand the structure and content of your webpage. This leads to better SEO rankings and improved accessibility for users with disabilities.

Common Semantic Elements:

Element Description
<header> Defines the header section of a document or a section, typically containing introductory content or navigation links.
<nav> Represents a section of a page that links to other pages or to parts within the page; a section with navigation links.
<main> Specifies the main content of the document, which is unique to the document and not repeated across pages (e.g., articles, blog posts).
<section> Defines a thematic grouping of content, typically with a heading. Used to break up content into logical sections.
<article> Represents a self-contained piece of content that could be independently distributed or reused (e.g., blog post, news article).
<aside> Defines content that is tangentially related to the content around it (e.g., sidebars, pull quotes).
<footer> Specifies the footer for a document or section, typically containing information about the author, copyright, links to related documents, etc.

Example Page Structure:

<html lang="en">
<head>
	<meta charset="UTF-8">
	<meta name="viewport" content="width=device-width, initial-scale=1.0">
	<title>Semantic HTML Example</title>
</head>
<body>
	<header>
		<h1>Website Title</h1>
		<nav>
			<ul>
				<li><a href="#">Home</a></li>
				<li><a href="#">About</a></li>
				<li><a href="#">Contact</a></li>

		</nav>
	</header>

		<section>
			<h2>Section Title</h2>

		</section>

			<p>This is a paragraph within an article.</p>

			<ul>
				<li><a href="#">Link 1</a></li>

</html>

Using semantic HTML elements helps create a well-structured, accessible, and SEO-friendly webpage.


üìÉ Div, Span, Class & ID

  1. The <div>Element

    2. The <div> element is a block-level container used to group other HTML elements together. It doesn't have any semantic meaning on its own but is useful for applying styles or scripts to a section of the page.A generic container for grouping content. It's a block-level element.

    Example:

    		
<div class="container">
	<h2>Section Title</h2>	
	<p>This is a paragraph inside a div.</p>
</div>

  2. The <span> Element

    3. The <span> element is an inline container used to group text or other inline elements. Like <div>, it has no semantic meaning but is useful for applying styles or scripts to a specific part of the text.A generic inline container for phrasing content. It's an inline element.

    Example:

    <p>This is a <span class="highlight">highlighted</span> word.</p>

  3. Class Attribute

    4. The class attribute is used to assign one or more class names to an HTML element. Classes are reusable and can be applied to multiple elements. They are commonly used for styling with CSS or selecting elements with JavaScript. For grouping multiple elements (reusable)

    Example:

    <div class="box highlight">This div has two classes.</div>
<span class="highlight">This span has one class.</span>

  4. ID Attribute

    5. The id attribute is used to assign a unique identifier to an HTML element. Each id must be unique within the document. IDs are often used for styling specific elements with CSS or for targeting elements with JavaScript. For identifying a single unique element

    Example:

    <div id="header">This div has a unique ID.</div>
<span id="highlighted-text">This span has a unique ID.</span>

    Key Differences Between Class and ID:

    Attribute Usage Uniqueness Selector in CSS
    class Used to group multiple elements for styling or scripting. Can be used on multiple elements. Prefixed with a dot (.) in CSS (e.g., .classname).
    id Used to uniquely identify a single element for styling or scripting. Must be unique within the document. Prefixed with a hash (#) in CSS (e.g., #idname).

    Use classes when you want to apply the same style or behavior to multiple elements. Use IDs when you need to target a specific, unique element.


üìëMulti-page Websites

Creating a Cohesive Website

A multi-page website consists of several interconnected HTML pages. Each page can have its own content but shares a common structure and navigation system to provide a seamless user experience.

Key Components of a Multi-page Website:

  1. Consistent Navigation

    2. Use a consistent navigation menu across all pages to help users easily find their way around the site. This can be achieved by including the same <nav> element in each HTML file or using server-side includes or templating engines.

    Example Navigation Menu:

    <nav>
	<ul>
		<li><a href="index.html">Home</a></li>
		<li><a href="about.html">About</a></li>
		<li><a href="services.html">Services</a></li>
		<li><a href="contact.html">Contact</a></li>
	</ul>
</nav>

  2. Relative Linking Between Pages

    3. Use relative file paths to link between different pages within your website. This ensures that links remain functional regardless of where the site is hosted.

    Example Links:

    	
<a href="about.html">About Us</a>
<a href="services.html">Our Services</a>
<a href="contact.html">Contact Us</a>

  3. Shared Resources

    4. Use a common set of CSS and JavaScript files across all pages to maintain a consistent look and feel. Link to these resources using relative paths in the <head> section of each HTML file.

    Example Resource Links:

    <link rel="stylesheet" href="css/styles.css">
<script src="js/script.js"></script>

  4. Footer Section

    5. Include a consistent footer on all pages with important information such as copyright notices, contact details, and links to privacy policies or terms of service.

    Example Footer:

    	
<footer>
	<p>¬© 2024 Your Company</p>
</footer>

By following these practices, you can create a well-structured multi-page website that provides a positive user experience and is easy to maintain.


Summary

Remember these key points:



  • Always use the HTML5 boilerplate as your starting point
  • Use relative paths for your own files, absolute paths for external resources
  • Structure your page with semantic HTML tags
  • Use classes for reusable styles and IDs for unique elements
  • Maintain consistent navigation across all pages of your website
